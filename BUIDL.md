# Seer - Wallet Intelligence & Autonomous Agent Layer for Mantle

Seer is an on-chain wallet intelligence and autonomous agent system built on Mantle. It gives users a live view of their portfolio risk and on-chain signals, then lets them act on those signals through natural-language intents, without giving up custody of their funds.

## What it does

Users connect their wallet and instantly get:

- A live portfolio summary with real balances pulled via Mantle RPC
- A wallet risk score and ranked identity profile generated by on-chain signal analysis
- Market signals driven by smart-money and token-flow data (via Nansen + DeFiLlama)
- An AI-powered reasoning layer (Claude) that explains _why_ signals matter, not just what they are

Users can then write intents in plain English:

> _"When mETH TVL climbs above $50M and my portfolio risk is below 60, commit up to 25 USDC weekly into the approved mETH strategy."_

Seer parses this into a structured execution policy (trigger type, allowed assets, protocols, spend cap, expiry), evaluates it against live conditions, and proposes a concrete transaction draft which the user signs. Seer relays signed transactions to Mantle RPC. Nothing moves without the user's signature.

## Background Workers

Active intents don't just sit idle. A background worker runs a fast job loop that continuously evaluates every active intent against live provider data. On each cycle it checks whether trigger conditions are met, builds an execution proposal if they are, and records a full execution log to Postgres covering condition results, actionability, transaction draft, and a reasoning hash. The same worker generates and persists signal snapshots and job-run summaries on each tick. A separate role (`APP_ROLE=worker`) lets the worker run independently from the API server, so intent evaluation scales without blocking user-facing requests.

## Mantle-native contracts (deployed & verified on Mantle Sepolia)

- `SeerArenaPoints`: point staking and settlement for the prediction arena
- `SeerPredictionRegistry`: on-chain prediction entry and resolution
- `SeerIdentitySBT`: non-transferable soul-bound identity token anchored to wallet reputation
- `SeerIntentRegistry`: on-chain anchoring of intent hashes and execution policy hashes

## Architecture highlights

- **Rust/Axum backend** with a provider abstraction over Nansen and DeFiLlama (with deterministic fallback so demos never break)
- **Four trigger modes**: Instant, Recurring, Conditional, RecurringConditional
- **ERC-4337 smart account path** for recurring/delegated execution via scoped session policies - spend limits, allowed assets, allowed protocols, expiry, and revocation are all enforced before any user operation is relayed
- **Claude as an explanation boundary**, not a source of financial facts - all signals come from RPC and provider data; Claude adds structured reasoning on top
- **PostgreSQL-backed durable state** for signals, intents, execution logs, and job runs - fully in-memory for local demos
- **Wallet auth** via EIP-191 challenge-sign-verify before any protected action

## Frontend

React/TypeScript with Privy wallet integration, live arena leaderboard, agent intent management, and real-time toast notifications linked to Mantle Sepolia Explorer transactions.
